Q1: When does Git perform fast-forward?

Fast-forward happens when all commits from the target branch are included in the branch being merged. Git does not have to merge any code, it just moves the pointer to the latest commit.

Q2: Merge vs Rebase?

Merge maintains the history and shows exactly when two branches were merged. Rebase rewrites history and creates a single line, which is easier to read but does not show exactly when two branches were merged.
